  - **Neo4j** `:7474`(Browser) / `:7687`(Bolt) — `RETURN 1` = 1, auth `neo4j/graphrag123`

### 1.2 Python 환경 (함정 ①)
- 시스템에 Python **3.14만** 설치됨 → ML 스택(llama-index/unstructured/ragas) 미지원.
- `winget install Python.Python.3.12`로 **3.12.10** 설치.
- venv 생성: `backend/.venv`, pip 26.1.2.

### 1.3 의존성 설치 + ragas 충돌 (함정 ②)
- `pip install -e backend` 1차 설치 후 import 검증: **19/20 통과, `ragas`만 실패**.
- 원인: `ragas 0.4.3`가 모듈 로드 시 `langchain_community.chat_models.vertexai` import →
  `langchain-community 0.4.0+`에서 해당 경로 삭제됨. ragas 0.4.x ↔ langchain 1.x 비호환.
- langchain은 **ragas만** 끌어옴(LlamaIndex는 langchain 불필요, 독립).
- 해결: `backend/pyproject.toml`에 핀 추가
  ```
  "ragas>=0.2.10,<0.3",
  "langchain-community<0.4",
  ```
  재설치 결과 `ragas 0.2.15` + `langchain-community 0.3.31` → **import 13/13 전부 통과**.

### 1.4 스모크 검증
- `app.main` import OK (`/health` 준비), `app.core.config.settings` 로드.
- Qdrant/Neo4j/Postgres 클라이언트 실연결 전부 응답.

## 2. PoC ① 멀티포맷 로더

### 2.1 샘플 문서 (가상 펌프 설비 매뉴얼 — 엔티티·관계·표 포함, 추후 그래프/멀티홉 재활용)
- `samples/pump_manual.md` — 직접 작성 (P-101, 모터 M-22, 베어링 B-7, 실 S-5 + 점검주기 표)
- `samples/pump_report.docx` — `scripts/make_samples.py`(python-docx) 생성 (P-202, 측정 표)
- `samples/pump_spec.pdf` — 같은 스크립트(PyMuPDF) 생성 (P-303, 영문)

### 2.2 로더 구현 (`backend/app/loaders/`)
- `base.py` — 공통 스키마 `Element`(type/content/location/meta), `LoadedDoc`(doc_id/doc_name/doc_type/file_path/elements + download_url), `make_doc_id()`(경로 sha1 12자리)
- `pdf_loader.py` — PyMuPDF. 페이지별 텍스트 + `find_tables()` 표 추출, 표→markdown
- `text_loader.py` — MD(헤딩 섹션 분할 + 표 라인 분리), TXT(offset)
- `unstructured_loader.py` — DOCX/XLSX/PPTX/HTML/EML, 표→HTML(`text_as_html`)
- `router.py` — 확장자 자동 분기 (PDF→PyMuPDF, MD/TXT→텍스트, 나머지→Unstructured)

### 2.3 검증 결과 (`scripts/poc1_loaders.py`)
| 파일 | 포맷 | 요소 | location | 표 인지 |
|---|---|---|---|---|
| pump_manual.md | md | 4 (text3+table1) | 섹션 헤딩 | MD표 보존 |
| pump_report.docx | docx | 7 (text6+table1) | `section:N` | HTML표 보존 |
| pump_spec.pdf | pdf | 1 (text1) | `p.1` | 아래 한계 |

- **위치 메타 보존** 확인 (MD=section, DOCX=section, PDF=page) → 출처 표기 기반 확보.
- 한계: 샘플 PDF 표는 `insert_text` 평문(격자선 없음) → `find_tables()` 미검출. 텍스트엔 포함. 격자 있는 실제 PDF는 검출됨. PoC엔 충분.

## 3. PoC ② 표 인지 청킹

### 3.1 구현 (`backend/app/chunking/`)
- `chunker.py` — `Chunk` 데이터클래스 + `chunk_document(doc)`
- 규칙:
  - **표 요소 = 항상 1청크, 무분할** (행·열 맥락 보존). 초대형 표만 행 분할하되 **헤더행을 각 조각에 반복** 삽입 → 조각 자기설명 유지.
  - **텍스트 요소** = 문장경계(`. ! ? 。 …` + 빈 줄) 기준 size 패킹, `MAX_CHARS=800`/`OVERLAP_CHARS=120`. 요소·location 경계 넘지 않음.
  - 모든 청크가 `doc_id/doc_name/doc_type/location/chunk_id/file_path/download_url` 전파.

### 3.2 검증 (`scripts/poc2_chunking.py`)
- 3개 샘플 청킹, 표 청크 구조(`|` 또는 `<table`) 무손상 assert.
- (실행 환경 메모) git-bash에서 Windows python + unstructured 동시 실행 시 **segfault** 발생 → PowerShell 또는 백그라운드 프로세스로 실행. 청킹 로직 자체는 정상.
