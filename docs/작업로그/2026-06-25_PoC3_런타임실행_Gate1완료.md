## 2. 실행 중 발견·수정한 버그 2건 (둘 다 실제 버그, 커밋 대상)

### 2.1 `backend/app/core/config.py` — `.env` 경로 (P0)
- **증상**: 루트에서 `scripts/poc3_graph.py` 실행 시 `ANTHROPIC_API_KEY` placeholder 오판 → RuntimeError. (backend/ 에서 실행하면 정상 → cwd 의존 버그)
- **원인**: `SettingsConfigDict(env_file=".env")` 상대경로. 프로세스 cwd가 루트면 루트의 `.env`(없음)를 찾아 키가 빈값.
- **수정**: `config.py` 파일 위치 기준 절대경로로 고정.
  ```python
  _ENV_FILE = Path(__file__).resolve().parents[2] / ".env"  # backend/.env
  model_config = SettingsConfigDict(env_file=str(_ENV_FILE), extra="ignore")
  ```
  → 어느 디렉터리에서 실행해도 `backend/.env` 읽음.

### 2.2 `backend/app/graph/extractor.py` — llama-index tool_choice 버그 우회 (P0)
- **증상**: 추출 단계에서 `anthropic.BadRequestError 400 - 'tool_choice: Input should be an object'`.
- **원인 (확정, 오프라인 재현)**: `llama-index-core 0.14.22` ↔ `llama-index-llms-anthropic 0.11.6`(**현 최신, 업그레이드 불가**) 상호작용 버그.
  - `structured_predict` 경로의 `FunctionCallingProgram` 기본 `tool_choice=None` → `apredict_and_call → achat_with_tools → _prepare_chat_with_tools`까지 `**kwargs`로 흘러감.
  - `_prepare_chat_with_tools` 최종 반환 `{..., **tool_choice_dict, **kwargs}`에서 `**kwargs`(`tool_choice=None`)가 **뒤에** 펼쳐져 올바르게 계산된 객체를 null로 덮어씀 → API에 null 전송.
- **수정**: 라이브러리(.venv) 직접 패치 대신 우리 코드에 `Anthropic` 서브클래스 추가. `_prepare_chat_with_tools`에서 비-dict `tool_choice`를 제거 → 부모가 계산한 객체(`{type:"any"}`) 보존. venv 재설치에도 살아남음.
  ```python
  class Anthropic(_Anthropic):
      def _prepare_chat_with_tools(self, *args, **kwargs):
          if not isinstance(kwargs.get("tool_choice"), dict):
              kwargs.pop("tool_choice", None)
          return super()._prepare_chat_with_tools(*args, **kwargs)
  ```
- **검증**: 오프라인으로 수정 전 `tool_choice=None`(BUG) → 수정 후 `{'type':'any'}`(FIXED) 확인. 이후 실제 실행 성공.

## 7. 함정 / 메모
1. **콘솔 한글 깨짐 ≠ 데이터 깨짐.** git-bash 한글 출력은 `PYTHONIOENCODING=utf-8` 주거나 Neo4j Browser로 확인. 저장값은 정상 UTF-8.
2. **tool_choice 버그는 LLM 호출 전 경로 전반에 영향.** W2 이후 다른 곳에서 structured 호출 추가 시 같은 `extractor.Anthropic` 서브클래스(또는 동등 우회) 재사용할 것. 라이브러리 자체는 0.11.6이 최신이라 업그레이드로 못 고침.
3. `config.py`는 이제 cwd 무관하게 `backend/.env` 읽음. 루트/backend/ 어디서 실행해도 동일.
4. PoC3 실행 셸: 기존 로그는 PowerShell 권장(unstructured/torch segfault 회피). `.md` 로더는 unstructured 미사용이라 git-bash에서도 이번에 정상 실행됨. DOCX/PDF 추출 단계는 PowerShell 유지 권장.
