## 2. PoC ③ 지식 그래프 추출/적재 (P0)

### 2.1 구현 (`backend/app/graph/`)
- `extractor.py` — LlamaIndex `SchemaLLMPathExtractor` + Claude(`llama_index.llms.anthropic.Anthropic`)
  - **열린 스키마**(`strict=False`): 도메인 엔티티/관계는 환각 억제용 **힌트**일 뿐 제한 아님.
    - 엔티티 힌트: 설비/펌프/모터/베어링/기계식실/열교환기/구성요소/점검주기/담당팀/절차
    - 관계 힌트: 구동/지지/연결/구성/정지/점검/담당/교체/주기
  - `build_llm()`이 placeholder 키(`sk-xxx`/빈값) 감지 시 즉시 RuntimeError.
- `store.py` — Neo4j 적재 + 인덱스 빌더
  - `chunks_to_nodes()`: `Chunk` → `TextNode`, **`chunk_id`를 노드 id로** 사용 → 그래프↔벡터 양방향 ID 연결 키 확보.
  - 출처 역링크: 소스(청크) 노드에 `doc_id/doc_name/doc_type/location/chunk_id/is_table/download_url` 메타 저장. 엔티티는 LlamaIndex `MENTIONS`로 청크 역참조.
  - `build_index()`: **`embed_kg_nodes=False`** → 임베딩 생략 → **OPENAI 키 불필요**(현재 placeholder 회피). `use_async=False`로 Windows 결정성 확보.
- `scripts/poc3_graph.py` — load→chunk→추출→Neo4j 적재 러너. 적재 후 `structured_query`로 노드/관계 수 + 샘플 트리플릿 25개 출력. 관계 0건이면 과소추정 경고.

### 2.2 설계 결정 근거
- **임베딩 분리**: PoC③ 목표는 1건 end-to-end 그래프 연결이지 검색 품질 아님 → 벡터/임베딩은 W2로 미룸. OPENAI 키 블로커 제거.
- **chunk_id = 노드 id**: W2 Qdrant 벡터와 동일 키로 조인 → 하이브리드 검색의 그래프-벡터 연결 기반.
- **열린 스키마**: 관계 과소추정이 단골 함정 → 스키마로 막지 않고 힌트로만 유도.
