## 1. P0 — 지금 고쳐야 할 정확성 결함

### 1.1 청킹 (`backend/app/chunking/chunker.py`)

- **[#1] 긴 단일 문장이 하드분할 안 됨** — `_split_text` (L43–63)
  - 문장 경계가 *0개일 때만* 하드 윈도우(L49–50). 경계가 있어도 패킹 후 한 조각이 `max_chars`(800)를 넘으면 그대로 통과한다. 한국어 긴 단락/CJK 런에서 재현.
  - 결과: Claude `max_tokens=2048`/`max_triplets_per_chunk=12` 초과 → 관계 누락 → 답 "없음".
  - 수정: 패킹 후에도 `len(chunk) > max_chars`인 조각에 하드 윈도우 재적용.

- **[#9] overlap seed가 `max_chars` 위반** — `_split_text` (L55–58)
  - 상한 가드가 문장 추가 *전*에만 동작. 다음 청크 seed `(tail + " " + s)`는 상한 미검사 → 822 > 800 재현.
  - 수정: seed 생성 후 길이 재검사, 초과 시 분할.

- **[#11] overlap 생자르기 → 단어 중간 조각** — `_split_text` (L57)
  - `cur[-overlap:]` 120자 raw 슬라이스를 다음 청크 머리에 붙임 → 음절 중간 끊긴 잡음. ~15% 토큰 낭비.
  - 수정: 마지막 *완결 문장*을 overlap으로 사용(이미 `sents` 보유).

- **[#10] 표 행패킹이 조인 `\n` 길이 누락** — `_split_table` (L77–83)
  - `cur_len = sum(len(x) for x in cur)`이 `"\n".join` 줄바꿈 글자를 빼고 계산 → 표 조각도 `max_chars` 초과.
  - 수정: 길이 계산에 줄 수만큼 `\n` 포함.

- **[#12] 빈 표 → 빈 청크** — `_split_table` (L66–67)
  - `_split_table("")` → `[""]` 반환 → Neo4j에 빈 source 노드 오염(text 경로는 빈 건 버림).
  - 수정: 빈 content 청크 생성 스킵.

## 2. P1 — 설계 약점 (결정 필요, W2~W3 전 합의)

- **[#6] 섹션별 청킹 → 멀티홉 깨질 위험** — `chunker.chunk_document` + `text_loader`
  - heading마다 청크가 갈린다. 표(3절)·절차(4절)가 다른 청크에 있으면 `SchemaLLMPathExtractor`는 청크 내부만 추출 → 멀티홉(`eval/dataset.jsonl` m1/m2/m3)은 **양쪽 청크가 같은 엔티티명**(`모터 M-22` vs `M-22`)을 뱉어 Neo4j에서 병합돼야만 성립한다. 이름이 갈리면 노드 분리 → 답 없음.
  - 결정지: (a) 엔티티 정규화/별칭 병합, (b) 청크 간 관계 룰기반 보강(W3), (c) 인접 섹션 슬라이딩 윈도우.

- **[#4] 마크다운 제목 텍스트가 본문 어디에도 없음** — `text_loader.load_markdown` (L44–48)
  - heading 매치 시 `section`만 세팅하고 제목 텍스트는 `buf`에 안 넣는다 → "3. 점검 주기" 같은 제목이 어느 청크 본문에도 없다. `eval/dataset.jsonl`이 그 문구를 `source.location`으로 쓰는데 추출/검색에서 매칭 불가.
  - 수정: 제목 줄을 다음 블록 첫 줄로 포함하거나 해당 Element에 제목 프리픽스.

- **[#7] 출처가 `section:0`으로 뭉개짐** — `unstructured_loader._location` (L25–26)
  - `page_number` 없는 DOCX/HTML 다수가 같은 `section:0`. UI 출처 표기·다운로드 연동이 무의미해진다.
  - 수정: heading 경로/요소 인덱스 기반 고유 location 생성.

- **[#8] TXT 청크 전부 `offset:0`** — `text_loader.load_text` (L58)
  - 긴 `.txt`가 N청크로 쪼개져도 모두 `offset:0`. `base.py`가 약속한 offset 추적 미구현.
  - 수정: 청킹 시 누적 char offset 계산해 청크별 location 부여.
