# PDF-RAG: 하이브리드 RAG 기반 설비 매뉴얼 및 SOP 지식 관리 시스템

> **AI 포트폴리오 프로젝트** (1인 개발, 12주)
> **핵심 전략:** 단순 챗봇을 넘어선 **"질문 → 답변 → 출처 하이라이트 관통 데모 및 RAGAS 정량 평가 검증"** 시스템

---

## 🚀 프로젝트 개요
수천 페이지에 달하는 설비 매뉴얼(도면, 표, 수치 혼재)과 SOP(표준작업절차서)는 현장 엔지니어의 신속한 장애 조치를 방해하는 병목이 됩니다. 본 프로젝트는 **정밀한 OCR/레이아웃 분석**과 **하이브리드 검색(Keyword + Vector)** 및 **리랭킹** 기술을 결합하여, 자연어 질문에 대해 **정확한 출처(페이지 및 도면 내 바운딩 박스)를 시각화**하여 제시하는 고신뢰성 B2B 솔루션입니다.

---

## ✨ 핵심 기능 (Key Features)
1. **정밀 데이터 파싱 & OCR**
   * 복잡한 표/도면의 구조적 정보 보존 및 계층적 청킹 (Hierarchical Chunking)
   * OCR 단계부터 바운딩 박스(Bounding Box) 좌표 메타데이터를 유지하여 UI 단으로 전송
2. **하이브리드 검색 & 리랭킹**
   * 키워드 검색(BM25)과 밀집 벡터 검색(Dense Vector)의 결합
   * BGE-Reranker를 통한 검색 결과 재정렬로 환각 현상(Hallucination) 최소화
3. **스플릿 뷰 & 바운딩 박스 하이라이팅 UI**
   * 답변과 PDF 원본을 동시에 제공하는 듀얼 스플릿 뷰
   * PDF.js를 활용하여 답변의 근거가 된 도면/텍스트의 정확한 좌표에 하이라이트 박스 표시
4. **RAGAS 기반 자동 평가 루프**
   * Hit Rate, MRR, 환각률, 응답 속도 정량 측정

## 🛠 기술 스택 (Tech Stack)

| 구분 | 기술 스택 | 설명 |
|---|---|---|
| **Frontend** | `Next.js`, `TypeScript`, `Tailwind CSS`, `PDF.js` | 스플릿 뷰 및 좌표 매핑 하이라이트 구현 |
| **Backend** | `FastAPI`, `PostgreSQL` | 고성능 비동기 API 서버 및 메타데이터 관리 |
| **Parsing & OCR** | `PyMuPDF`, `Unstructured`, `LayoutLMv3` | 텍스트/표 파싱 및 좌표 데이터 보존 |
| **Vector DB** | `Qdrant` | 가볍고 빠른 싱글 노드/PoC용 벡터 데이터베이스 |
| **Framework** | `LlamaIndex` | RAG 파이프라인 구축 및 데이터 인덱싱 일원화 |
| **LLM / Reranker**| `Solar LLM`, `BGE-Reranker` | 국산 한국어 특화 LLM 및 최적 컨텍스트 선별 리랭커 |
| **Evaluation** | `RAGAS`, `DeepEval` | RAG 성능 지표 정량 평가 |
| **Infrastructure**| `Docker` | 개발/운영 환경 컨테이너화 및 배포 표준화 |

## 📅 개발 로드맵 (12주)
* **1~2주차: 설계 및 PoC** - 요구사항 정의, 메타데이터 설계, OCR bbox 보존 PoC 검증
* **3~6주차: 파이프라인 개발** - 파싱 및 OCR 고도화, 계층적 청킹 구현, Qdrant 벡터 인덱싱
* **7~8주차: RAG 엔진 구축** - 하이브리드 검색 결합, 리랭커 적용 및 프롬프트 튜닝
* **9~10주차: 프론트엔드 연동** - 듀얼 스플릿 뷰 구현, PDF 좌표 매핑 하이라이팅 UI 완성
* **11~12주차: 정량 평가 및 배포** - RAGAS 평가 자동화, 현장 시나리오 테스트, Docker 패키징
