# RAG 챗 UI 프론트엔드 개발 계획서 (ChatGPT/Gemini 스타일)

본 계획서는 앞서 구축한 **DB 명세서 특화 RAG 백엔드**를 사용자가 쉽게 활용할 수 있도록, **ChatGPT·Gemini 스타일의 대화형 웹 인터페이스**를 구축하기 위한 프론트엔드 기획안입니다. 핵심은 (1) 파일 업로드, (2) 깔끔한 그리드 기반 레이아웃, (3) 질문-답변 대화창 입니다.

---

## 1. 목표 및 핵심 컨셉 (Concept)

* **익숙한 UX**: 사용자가 ChatGPT/Gemini를 쓰듯 별도 학습 없이 질문을 입력하고 답변을 받는 경험 제공.
* **문서 업로드 → 즉시 질의**: DB 명세서(MD/PDF 등)를 화면에서 드래그 앤 드롭으로 업로드하면 백엔드 RAG 파이프라인이 자동 인덱싱 후 바로 질문 가능.
* **그리드 레이아웃**: 사이드바 + 메인 대화 영역 + (선택) 출처/문서 패널의 **반응형 그리드 구조**로 정보를 깔끔하게 배치.

---

## 2. 화면 레이아웃 설계 (Grid Layout)

```
┌──────────────┬───────────────────────────────┬──────────────┐
│  사이드바     │        메인 대화 영역          │  출처 패널    │
│ (Sidebar)    │     (Chat Area)               │ (Sources)    │
│              │                               │              │
│ + 새 대화     │  ┌─────────────────────────┐  │  검색된      │
│ 대화 기록     │  │ 🧑 질문 말풍선          │  │  테이블/청크 │
│ ─────────    │  │ 🤖 답변 말풍선 (DDL 등) │  │  미리보기    │
│ 업로드 문서   │  └─────────────────────────┘  │              │
│ 목록         │                               │  복사 버튼    │
│              │  ┌─────────────────────────┐  │              │
│ ⚙ 설정       │  │ 📎 입력창 + 전송 버튼   │  │              │
└──────────────┴───────────────────────────────┴──────────────┘
```

### 2.1 사이드바 (좌측)
* **새 대화 시작** 버튼, **대화 기록(세션) 목록**, **업로드된 문서 목록** 표시.
* 문서별 인덱싱 상태(처리 중 / 완료) 뱃지 표시.

### 2.2 메인 대화 영역 (중앙)
* 질문/답변을 **말풍선(채팅 버블)** 형태로 위→아래 스택.
* AI 답변은 **마크다운 렌더링**(코드블록, 표, DDL 하이라이팅) 지원, 메시지마다 **복사 버튼**.
* 하단 고정 **입력창**: 멀티라인 입력, `Enter` 전송 / `Shift+Enter` 줄바꿈, 📎 파일 첨부 아이콘.
* **스트리밍 출력**: 답변이 토큰 단위로 타이핑되듯 표시(typing effect).

### 2.3 출처 패널 (우측, 토글 가능)
* 답변 생성에 사용된 **검색 청크(테이블/컬럼)** 출처를 카드로 표시 → 신뢰성 확보.
* 출처 카드 클릭 시 원문 문서 위치로 점프.

## 4. 추천 기술 스택 (Tech Stack)

| 영역 | 기술 스택 | 선정 이유 |
| :--- | :--- | :--- |
| **프레임워크** | `Next.js (React)` + `TypeScript` | SSR/라우팅 내장, 채팅 UI 컴포넌트 생태계 풍부, 스트리밍 응답 처리 용이 |
| **스타일링** | `Tailwind CSS` + `shadcn/ui` | 그리드 레이아웃·다크모드·컴포넌트 빠르게 구성, 일관된 디자인 |
| **상태 관리** | `Zustand` 또는 `React Query` | 대화 세션·서버 응답 캐싱 관리 경량화 |
| **마크다운 렌더링** | `react-markdown` + `react-syntax-highlighter` | DDL/코드/표 안전 렌더링 및 구문 강조 |
| **스트리밍** | `Server-Sent Events (SSE)` / `fetch stream` | 토큰 단위 타이핑 효과 구현 |
| **백엔드 연동** | `FastAPI` (RAG 엔진 래핑) | 2.x 계획서의 RAG 파이프라인을 REST/SSE API로 노출 |

> **초단기 대안**: 백엔드 개발자 1인 PoC라면 `Streamlit` 또는 `Gradio`의 `ChatInterface`로 1일 만에 챗+업로드 UI 프로토타입 가능. 본격 제품화 시 Next.js로 전환 권장.

## 5. API 연동 설계 (Backend Contract)

RAG 백엔드(FastAPI)가 제공해야 할 최소 엔드포인트:

```
POST /api/upload        # 파일 업로드 → 인덱싱 작업 큐 등록, job_id 반환
GET  /api/documents     # 업로드/인덱싱된 문서 목록 및 상태 조회
POST /api/chat          # { query, session_id } → 답변 스트리밍(SSE) + 출처(sources)
GET  /api/sessions      # 대화 세션 목록
DELETE /api/sessions/:id
```

응답 예시 (`/api/chat` SSE 이벤트):
```json
{ "type": "token", "content": "CREATE TABLE users (" }
{ "type": "sources", "data": [ { "table": "users", "score": 0.92 } ] }
{ "type": "done" }
```

## 6. 단계별 구축 로드맵 (2주 MVP)

### **[1주차] 레이아웃 & 기본 대화 흐름**
* **1~2일차**: Next.js + Tailwind + shadcn/ui 프로젝트 셋업, 3분할 그리드 레이아웃(사이드바/대화/출처) 골격 구현.
* **3~4일차**: 채팅 말풍선 컴포넌트, 입력창(멀티라인·전송), 마크다운/코드 렌더링 구현.
* **5일차**: FastAPI `/api/chat` 연동 및 SSE 스트리밍 타이핑 출력 구현.

### **[2주차] 업로드 & 세션 관리 마감**
* **6~7일차**: 드래그 앤 드롭 파일 업로드 UI + 인덱싱 진행률/상태 폴링 연동.
* **8일차**: 대화 세션 저장·전환·삭제, 사이드바 문서 목록 연동.
* **9일차**: 출처 패널, 복사 버튼, 다크모드 등 디테일 마감.
* **10일차**: 반응형 점검 및 실제 DB 명세서 업로드→질의 E2E 테스트.

## 7. 디자인 디테일 가이드

* **컬러/여백**: 중립 회색 배경 + 화이트 카드, 충분한 패딩으로 그리드 간 시각적 분리.
* **타이포그래피**: 본문은 가독성 좋은 sans-serif, 코드/DDL은 monospace.
* **빈 상태(Empty State)**: 첫 진입 시 "문서를 업로드하고 질문해보세요" 안내 + 예시 질문 칩(chip) 노출.
* **로딩**: 답변 생성 중 점멸 커서/스켈레톤으로 대기감 완화.
