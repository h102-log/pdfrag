# 하이브리드 RAG 기반 설비 매뉴얼 및 SOP 지식 관리 시스템 개발 계획서

> **포스코DX 산업 AI 포트폴리오 프로젝트** · 1인 개발 · 12주
> **핵심 전략:** 완벽한 OCR이 아니라 **"질문 → 답변 → 출처 하이라이트 관통 데모 + 정량 평가로 증명"**

---

## 1. 프로젝트 개요

### 배경
- 수천 페이지 설비 매뉴얼(도면·표·수치 혼재)로 현지 엔지니어의 장애 조치 및 지식 접근성 저하.
- 단순 PDF 챗봇의 환각(Hallucination) 현상으로 산업 현장 적용 불가.

### 목적
- 고성능 OCR + 하이브리드 검색을 결합, **정확한 출처(페이지·도면 위치)와 함께 자연어 조치 방법을 즉시 제공**하는 B2B 솔루션 구축.

### 차별화 포인트 (단순 RAG와의 구분선)
1. **출처 좌표 하이라이팅** — 답변 근거를 도면 바운딩 박스로 시각화 (이 프로젝트의 핵심 가치).
2. **RAG 평가 자동화 루프** — RAGAS 기반 정확도·환각률 정량 측정 및 개선 (4년차 차별화 무기).
3. **(확장) GraphRAG** — 설비–부품–에러코드 관계 그래프로 멀티홉 추론 (여유 시 추가).

> **포지셔닝:** "최신 기술 자랑"이 아니라 **"산업 현장에서 신뢰성 있게 돌아가는 것을 책임지고 만든 엔지니어"**. 기업이 경력직에게 기대하는 가치.

## 2. 핵심 기능 요건

### 2.1 정밀 데이터 파싱 & OCR  `리스크: 중상`
- `LayoutLMv3` 등으로 복잡한 표 구조를 마크다운으로 변환, 도면 내 텍스트/수치를 계층적 청킹(Hierarchical Chunking)으로 전처리.
- **필수 설계 원칙:** OCR 단계에서 추출한 **bbox 좌표를 메타데이터로 끝까지 보존** → 하이라이팅 연동의 생명줄.
- **현실 전략:** 파인튜닝 X, **사전학습 모델 추론 + `Unstructured`/`PyMuPDF` 우선 검토**. OCR 완벽주의 금지.

### 2.2 하이브리드 검색 & 리랭킹  `리스크: 낮음 (안전지대)`
- 키워드 검색(BM25, 설비코드·에러번호에 강함) + 벡터 검색(자연어 의도) 융합.
- Reranker(BGE-Reranker)로 최적 컨텍스트 선별 후 LLM 주입.

### 2.3 UI 스플릿 뷰 & 하이라이팅  `리스크: 중`
- PDF.js로 답변 근거가 된 정확한 페이지·도면 위치를 **바운딩 박스로 하이라이트**.
- 2.1의 bbox 좌표 전파가 선행 조건.

### 2.4 RAG 평가 자동화  `신규 · 차별화`
- RAGAS/DeepEval로 **Hit Rate, MRR, 환각률, 응답속도** 측정.
- 측정 → 개선 → 재측정 루프를 문서화 (의사결정 로그).

## 3. 추천 기술 스택  *(과잉 제거 · 1인 PoC 최적화)*

| 영역 | 선정 | 비고 |
|------|------|------|
| 프론트엔드 | Next.js / TypeScript, Tailwind CSS, **PDF.js** | 좌표 매핑·하이라이팅 |
| 백엔드 | **FastAPI** (Python), PostgreSQL | 표준·검증됨 |
| 데이터 전처리 | **PyMuPDF, Unstructured** (+ LayoutLMv3 추론) | 파인튜닝 X |
| 벡터 DB | **Qdrant** | 단독 PoC에 가벼움 (Milvus는 대규모 클러스터용 → 제외) |
| RAG 엔진 | **LlamaIndex** | LangChain과 병기 X, 하나로 통일 |
| LLM | **Solar** | 한국어·국산, 포스코DX 어필 |
| 리랭커 | BGE-Reranker | |
| 평가 | **RAGAS / DeepEval** | 정량 지표 |
| 인프라 | **Docker** | 엔터프라이즈·운영 어필 |

> **택1 결정 (면접 대비):** LangChain/LlamaIndex → **LlamaIndex** · Milvus/Qdrant → **Qdrant**. "왜 둘 다?" 질문 차단.

## 4. 개발 일정 (총 12주) — *MVP 관통 우선 배분*

| 주차 | 단계 | 핵심 산출물 |
|------|------|-------------|
| **1~2주** | 설계 + PoC | 요구사항·메타데이터 구조·아키텍처 설계, **OCR PoC 1건**(bbox 좌표 보존 검증) |
| **3~6주** | 파이프라인 | OCR/파싱 **4주**(리스크 반영), 계층적 청킹 1주, Qdrant 인덱싱 |
| **6~8주** | RAG 엔진 | 하이브리드 검색·리랭커, 프롬프트 엔지니어링, 환각 제어 |
| **9~10주** | 프론트엔드 | 스플릿 뷰 + PDF 좌표 연동 하이라이트 UI |
| **11~12주** | 평가·안정화 | **RAGAS 정량 평가**, 현장 시나리오 QA, Docker 패키징·배포 |

### 일정 리스크 관리
- **함정 구간:** 3~6주 OCR. 도면 OCR + bbox 좌표 보존은 과소 추정 단골 → **4주 확보**.
- **관통 우선:** 품질보다 **end-to-end(질문→답변→출처 하이라이트) 먼저 연결** 후 깊이 추가.
- **범위 축소(MVP):** 매뉴얼 1~2종, 표/도면 일부 유형으로 한정.

## 5. 기대 효과

### 정성
- 복잡한 설비 장애 조치 시간(MTTR) 단축 및 휴다운타임 최소화.
- 현장 엔지니어 간 지식 상향 평준화.
- 포스코DX 고부가가치 B2B AI 솔루션 패키지 자산화.

### 정량 (포트폴리오 증명 지표)
- RAG 정확도: **Hit Rate, MRR**
- 신뢰성: **환각률(%) 감소폭**
- 성능: **평균 응답속도(s)**

> 막연한 "단축"이 아니라 **본인 시스템의 측정값·개선폭**으로 제시 → "측정하고 개선한 엔지니어" 증명.

## 6. 포스코DX 이직 어필 포인트 (4년차 차별화)

1. **정량 지표 제시** — 측정→개선 서사로 "구현자"가 아닌 "개선자" 포지셔닝.
2. **트레이드오프 의사결정 로그** — "왜 Qdrant·이 청킹·이 리랭커?" → 면접 질문 자판기.
3. **도메인 이해도** — 설비코드/에러번호 체계를 현실적으로 반영.
4. **MLOps 인상** — Docker + 평가 자동화로 "운영까지 본다".
